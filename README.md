# Poker Stats Bot

Telegram-бот для сбора и отображения статистики входов/выходов участников в групповом чате. Удобен для подсчёта фишек в покере, долгов или любых других единиц с разделением на операции «Вход» и «Выход».

Дополнительно предоставляет HTTP API для получения статистики, что позволяет интегрировать веб‑интерфейс или другие инструменты. В состав монорепозитория входит готовый React‑фронтенд для визуализации данных.

---

## 🚀 Возможности

- Парсинг сообщений вида `+<сумма> | <ник>` внутри секций `Вход:` и `Выход:`.
- Сохранение каждой транзакции в базе данных SQLite (`better-sqlite3`).
- Подсчёт суммарных входов, выходов и разницы для каждого участника.
- Команды для просмотра статистики прямо в чате (`/stats`, `/top`, `/help`, `/stats_update`).
- Автоматическое удаление ответов бота через 30 секунд (опционально).
- Удаление сообщений пользователей с командами (для чистоты чата).
- Поддержка даты игры (можно указать через `game DD.MM.YYYY` в сообщении).
- HTTP API для получения статистики в формате JSON.
- Гибкая структура кода, разделённая на слои (репозитории, сервисы, обработчики).
- Веб‑интерфейс на React (опционально) для просмотра статистики через браузер.

## 🛠 Технологии

- **Backend:** Node.js + TypeScript, [Telegraf](https://telegraf.js.org/), [better-sqlite3](https://github.com/WiseLibs/better-sqlite3), [Express](https://expressjs.com/), [pino](https://getpino.io/)
- **Frontend:** React + TypeScript, Vite
- **Монорепозиторий:** pnpm workspaces
- **Тестирование:** Vitest
- **Контейнеризация:** Docker, Docker Compose

## 📦 Быстрый старт

### Требования

- [Node.js](https://nodejs.org/) версии 18 или выше
- [pnpm](https://pnpm.io/) версии 8 или выше
  *(Установить можно командой `corepack enable && corepack prepare pnpm@latest --activate`)*
- Токен бота от [@BotFather](https://t.me/botfather)

### Установка и запуск в режиме разработки

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/yourusername/poker-stats-bot.git
   cd poker-stats-bot
   ```

2. **Установите зависимости:**

   ```bash
   pnpm install
   ```

3. **Создайте файл с переменными окружения:**
   - Для локальной разработки скопируйте `.env.dev` в `.env` (или просто переименуйте).

   ```bash
   cp .env.dev .env
   ```

   - Отредактируйте `.env`, указав токен бота:

   ```env
   BOT_TOKEN=ваш_токен_бота
   # TELEGRAM_API_URL=  # опционально, если используете локальный Telegram API
   # API_PORT=3000       # порт для HTTP API (по умолчанию 3000)
   ```

4. **Запустите проект в dev-режиме:**

   ```bash
   # Запускает и бэкенд, и фронтенд одновременно
   pnpm run dev

   # Или по отдельности:
   pnpm run dev:backend   # только бэкенд (порт 3000)
   pnpm run dev:frontend  # только фронтенд (порт 5173)
   ```

   После запуска:
   - Бэкенд-сервер с ботом и API будет доступен на `http://localhost:3000`
   - Фронтенд (если запущен) — на `http://localhost:5173` (он автоматически проксирует запросы к `/api` на бэкенд)

5. **Проверьте работу бота:**
   - Добавьте бота в группу Telegram.
   - Дайте ему права администратора (хотя бы на удаление сообщений).
   - Отправьте команду `/help` в чат.

### Сборка и запуск в production (без Docker)

```bash
# Сборка всех пакетов
pnpm run build

# Запуск скомпилированного бэкенда
pnpm run start
```

Собранный бэкенд будет лежать в `packages/backend/dist`. База данных сохраняется в папке `data/` в корне проекта.

### Тестирование

```bash
# Запуск тестов во всех пакетах
pnpm run test
```

## ⚙️ Конфигурация окружения

| Файл       | Назначение                                                                                      |
| ---------- | ----------------------------------------------------------------------------------------------- |
| `.env.dev` | Пример конфигурации для разработки. Содержит тестовые значения (не используется в production).  |
| `.env`     | Основной файл переменных окружения. Используется при запуске через `pnpm run start` или Docker. |

**Важно:** в production (Docker) бот по умолчанию ищет файл `.env` внутри контейнера по пути `/app/.env`. Рекомендуется передавать переменные через `environment` в `docker-compose.yml` или через секреты оркестратора.

Доступные переменные:

- `BOT_TOKEN` — **обязательно**, токен Telegram‑бота.
- `TELEGRAM_API_URL` — опционально, URL локального Bot API.
- `API_PORT` — порт для HTTP API (по умолчанию `3000`).
- `LOG_LEVEL` — уровень логирования (`info`, `debug`, `error`).

## 🤖 Команды бота

| Команда         | Описание                                                                                                                                                                  |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/stats`        | Показать подробную таблицу статистики (входы, выходы, разница). Поддерживает фильтры: `/stats all` (всё время), `/stats 2024` (конкретный год), `/stats` (последний год). |
| `/top`          | Топ‑10 участников по разнице (выход минус вход). Поддерживает те же фильтры, что и `/stats`.                                                                              |
| `/stats_update` | Принудительный пересчёт агрегированной статистики (полезно после ручного изменения БД).                                                                                   |
| `/help`         | Показать справку по командам и формату сообщений.                                                                                                                         |

### Как добавлять данные

Сообщения должны содержать секции `Вход:` и `Выход:` со строками вида `+<сумма> | <ник>`. Комментарии после `//` игнорируются.

**Пример текстового сообщения:**

```
Вход:
+500 | Тема
+700 | @Rabotyaga3000
Выход:
+1840 | @EgorVaganov1111
+290 | @kovass
```

**Пример с указанием даты игры (упоминание бота):**

```
@your_bot game 16.02.2025
Вход:
+500 | Тема
+700 | @Rabotyaga3000
Выход:
+1840 | @EgorVaganov1111
```

**Фото с подписью** также поддерживается (подпись обрабатывается как текст).

**Редактирование сообщения** – если отредактировать сообщение, содержащее данные игры, бот обновит соответствующую запись (удалит старые транзакции и добавит новые).

## 🌐 HTTP API

Бот предоставляет REST API для получения статистики. По умолчанию API запускается на порту `3000` (можно изменить через `API_PORT` в `.env`).

### Endpoint

```
GET /api/stats?chatId=<ID_чата>&filter=<all|2024>
```

**Параметры:**

- `chatId` (обязательно) — числовой идентификатор чата.
- `filter` (опционально):
  - `all` – статистика за всё время
  - `2024` – статистика за указанный год (YYYY)
  - без параметра – статистика за последний год

**Пример ответа:**

```json
[
  {
    "username": "Тема",
    "total_in": 500,
    "total_out": 3500,
    "games_count": 3
  },
  {
    "username": "@kovass",
    "total_in": 290,
    "total_out": 290,
    "games_count": 1
  }
]
```

Эти данные используются фронтендом (папка `frontend`) для отображения таблицы.

## 🐳 Контейнеризация

В корне проекта находятся `Dockerfile` и `docker-compose.yml` для запуска бэкенда в изолированном окружении.

### Dockerfile (multi‑stage)

- **Stage 1 (deps):** устанавливает зависимости с помощью pnpm.
- **Stage 2 (builder):** компилирует TypeScript.
- **Stage 3 (pruned):** создаёт облегчённую продакшен‑папку (`pnpm deploy --legacy`).
- **Stage 4 (runner):** финальный образ на основе `node:18-slim`, копирует только необходимое.

### Запуск через Docker Compose

1. Убедитесь, что в корне есть файл `.env` с токеном бота.
2. Запустите контейнер:
   ```bash
   docker-compose up -d
   ```
3. Проверьте логи:
   ```bash
   docker-compose logs -f
   ```

База данных будет храниться в примонтированной папке `./data` на хосте, что обеспечивает сохранность данных между перезапусками.

### Сборка образа вручную

```bash
docker build -t poker-stats-bot .
docker run -d --name poker-bot -v $(pwd)/data:/app/data --env-file .env poker-stats-bot
```
